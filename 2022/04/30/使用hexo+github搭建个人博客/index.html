相关文件解释（可跳过）

_config.yml

  • 网站的 配置 信息，您可以在此配置大部分的参数。

package.json

  • 应用程序的信息。EJS, Stylus 和 Markdown renderer 已默认安装，您可以自由移除。

scaffolds

  • 模版 文件夹。当您新建文章时，Hexo 会根据 scaffold 来建立文件。

  • Hexo的模板是指在新建的文章文件中默认填充的内容。例如，如果您修改scaffold/post.md中的Front-matter内容，那么每次新建一篇文章时都会包含这个修改。

source

  • 资源文件夹是存放用户资源的地方。除 _posts 文件夹之外，开头命名为 _ (下划线)的文件 / 文件夹和隐藏的文件将会被忽略。Markdown 和 HTML 文件会被解析并放到 public 文件夹，而其他文件会被拷贝过去。

themes

  • 主题 文件夹。Hexo 会根据主题来生成静态页面。

在Hexo中启用Next主题

  • Blog根目录的_config.yml中修改

  • 将thems/next/_config.yml配置文件复制到Blog根目录下并改名为_config.next.yml

    不要直接修改thems/next/_config.yml文件，因为使用git对next主题进行更新的时候可能会覆盖掉_config.yml

    所以修改_config.next.yml为最佳

    具体可参考Configuration | NexT (theme-next.js.org)

发布到GitHub

GitHub中准备好建站仓库

  • 创建自己账户名对应的yourname.github.io仓库。

  • 查看自己仓库的默认分支是master还是main

  • 在_config.yml文件中添加或修改

  • 部署到仓库
